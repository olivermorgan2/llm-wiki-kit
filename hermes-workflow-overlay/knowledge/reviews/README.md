# reviews/

Verbatim adversarial-review artifacts, one file per review:
`YYYY-MM-DD-<reviewer>-<subject>.md` (e.g. `2026-07-01-codex-adr-001-005-review.md`).

Archived verbatim, never edited. Every ADR acceptance and phase closeout
must reference a file here — the guard workflow enforces that ADR PRs add
one. If the reviewer is unavailable, the phase halts; no receipt, no merge.
